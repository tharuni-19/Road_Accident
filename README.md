# Road Accident Analysis — Python + Power BI Project

An end-to-end AI & Data Science project: Python handles data cleaning,
feature engineering and EDA (with an optional ML model), and Power BI
turns the processed data into an interactive dashboard.

---

## 1. Project Structure

```
road_accident_project/
├── generate_dataset.py          # (optional) creates a synthetic dataset — skip if you have real data
├── 01_data_cleaning.py          # missing values, duplicates, dtype fixes
├── 02_feature_engineering.py    # date/time features, severity score, etc.
├── 03_eda.py                    # exploratory data analysis + charts
├── 04_predictive_modeling.py    # optional: Random Forest severity prediction
├── requirements.txt
├── data/
│   ├── raw_accident_data.csv
│   ├── cleaned_accident_data.csv
│   └── final_accident_data_for_powerbi.csv   ← load THIS into Power BI
└── charts/                      # PNG charts generated by 03_eda.py
```

## 2. How to Run the Python Pipeline

```bash
pip install -r requirements.txt

# Step 0 (only if you don't already have a dataset):
python generate_dataset.py

# Step 1: Clean the data
python 01_data_cleaning.py

# Step 2: Engineer features
python 02_feature_engineering.py

# Step 3: Exploratory Data Analysis
python 03_eda.py

# Step 4 (optional): Predictive modeling
python 04_predictive_modeling.py
```

**If you have a real dataset** (e.g. from Kaggle or a government open
data portal), just replace `data/raw_accident_data.csv` with your file.
Rename your columns to match the ones used in `01_data_cleaning.py`
(or edit the column names in that script), then run steps 1–4 as above.

### Final column list (loaded into Power BI)

| Column | Description |
|---|---|
| Accident_ID | Unique record ID |
| Date, Time | Original date/time |
| Year, Month, Month_Name, Day, Day_Name, Hour | Date parts |

| Vehicle_Type | Car, Truck, Two-Wheeler, etc. |
| Number_of_Vehicles_Involved | Count |
| Weather_Condition | Clear, Rainy, Foggy, etc. |
| Road_Type, Road_Surface_Condition, Light_Condition | Road conditions |
| Speed_Limit | km/h |
| Driver_Age, Age_Group, Driver_Gender | Driver info |
| Severity | Minor / Serious / Fatal |
| Fatalities, Serious_Injuries, Minor_Injuries, Total_Casualties | Casualty counts |
| Severity_Score | Weighted numeric severity (Fatal=5, Serious=2, Minor=1) |

## 3. Building the Power BI Dashboard

> Power BI Desktop is a GUI application, so the dashboard itself is
> built interactively — this section gives you the exact steps and
> DAX code to reproduce it.

### Step 1 — Import data
1. Open **Power BI Desktop** → `Get Data` → `Text/CSV`
2. Select `data/final_accident_data_for_powerbi.csv`
3. Click **Transform Data** to open Power Query, verify:
   - `Date` is typed as **Date**
   - `Is_Weekend`, `Is_Fatal_Accident` are typed as **True/False** or **Whole Number**
   - `Latitude` / `Longitude` are **Decimal Number**
4. Click **Close & Apply**

### Step 2 — Build a Date table (recommended)
Model view → `New Table`:
```DAX
DateTable = CALENDAR(MIN('final_accident_data_for_powerbi'[Date]), MAX('final_accident_data_for_powerbi'[Date]))
```
Then mark it as a **Date Table** (Table tools → Mark as Date Table) and
relate it to the `Date` column in your main table.

### Step 3 — Create DAX Measures
Go to **Modeling → New Measure** and add each of these:

```DAX
Total Accidents = COUNTROWS('final_accident_data_for_powerbi')

Fatal Accidents = CALCULATE([Total Accidents], 'final_accident_data_for_powerbi'[Severity] = "Fatal")

Serious Accidents = CALCULATE([Total Accidents], 'final_accident_data_for_powerbi'[Severity] = "Serious")

Minor Accidents = CALCULATE([Total Accidents], 'final_accident_data_for_powerbi'[Severity] = "Minor")

Total Fatalities = SUM('final_accident_data_for_powerbi'[Fatalities])

Total Serious Injuries = SUM('final_accident_data_for_powerbi'[Serious_Injuries])

Total Minor Injuries = SUM('final_accident_data_for_powerbi'[Minor_Injuries])

Total Casualties = SUM('final_accident_data_for_powerbi'[Total_Casualties])

Fatality Rate % = DIVIDE([Fatal Accidents], [Total Accidents], 0) * 100

Avg Casualties per Accident = DIVIDE([Total Casualties], [Total Accidents], 0)

Weekend Accidents = CALCULATE([Total Accidents], 'final_accident_data_for_powerbi'[Is_Weekend] = TRUE)

YoY Accident Change % =
VAR CurrentYear = [Total Accidents]
VAR PreviousYear = CALCULATE([Total Accidents], SAMEPERIODLASTYEAR('DateTable'[Date]))
RETURN DIVIDE(CurrentYear - PreviousYear, PreviousYear, 0) * 100
```

### Step 5 — Add Interactive Filters (Slicers)
Add slicer visuals for:
- `Year` / `Month_Name` (date range slicer)
- `State`
- `Severity`
- `Vehicle_Type`
- `Weather_Condition`
- `Time_of_Day`

Sync slicers across pages via **View → Sync Slicers** if you want
filters to apply on every page.

### Step 6 — Formatting tips
- Use a consistent color theme: red/orange tones for severity (Fatal =
  dark red, Serious = orange, Minor = yellow)
- Enable **drill-through**: right-click `State` bar → Add drillthrough
  page showing that state's accident details
- Add a **tooltip page** for the map showing a mini bar chart of
  severity breakdown per state
- Pin key KPI cards to the top of every page for consistency

## 5. Notes on the Sample Data
`generate_dataset.py` creates a **synthetic** dataset (6,000 rows) with
realistic messiness (missing values, duplicates, inconsistent casing,
invalid ages) so the cleaning pipeline has something real to fix. It's
meant purely to make this project runnable end-to-end without needing
you to source a real dataset first. Swap in your real dataset (Kaggle's
"Road Accident" / "US Accidents" datasets are common choices) whenever
you're ready — the rest of the pipeline works unchanged as long as
column names match (or you edit `01_data_cleaning.py`).
